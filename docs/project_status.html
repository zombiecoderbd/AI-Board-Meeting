    Meeting = Session + Multiple Conversations

  • ২
    Model Selection Page
    Admin থেকে Ollama এবং Gemini-র মডেল লিস্ট দেখা যাবে, একটি ডিফল্ট মডেল সিলেক্ট করা যাবে।
  • ৩
    Agent Persona Page
    Admin থেকে agent-এর নাম, role, system prompt কাস্টমাইজ করা যাবে। ZombieCoder identity সব agent-এ shared থাকবে।
  • ৪
    Settings Page
    AI Provider switch (Ollama/Gemini), audio settings, BD-context on/off, ইত্যাদি।
  • ৫
    Meeting UI — আধুনিক ডিজাইন
    "ফিল মিটিং ফিল" — যে বলছে তার indicator জ্বলবে, বাংলা ভয়েস natural শোনাবে।
  • ৬
    বাংলাদেশ মার্কেট বেইজড বিশ্লেষণ
    প্রতিটি agent-এর system prompt-এ BD-specific context থাকবে।

৩. আমার মন্তব্য (Technical Assessment)

Core architecture ঠিক আছে। Next.js + TypeScript + SQLite + WebSocket — এই stack টা solid। কিন্তু এটা এখন একটি single-page prototype, পূর্ণ product না।

Session management DB-তে আছে কিন্তু সেটা শুধু background-এ। UI-তে পুরানো meeting resume করার কোনো flow নেই — এটাই সবচেয়ে বড় গ্যাপ।

Ollama দিয়ে local model চালানো হচ্ছে, কিন্তু gemma2:2b বাংলায় দুর্বল। Gemini Flash API key কনফিগে আছে কিন্তু ব্যবহার হচ্ছে না — এটা activate করলে বাংলা quality নাটকীয়ভাবে বাড়বে।

বাংলা TTS-এর জন্য Browser Web Speech API যথেষ্ট না। Google TTS (বিনামূল্যে tier আছে) অথবা Coqui TTS (open-source, local) দিয়ে natural বাংলা voice সম্ভব।

⚠️
.env ফাইলে Gemini API Key সরাসরি লেখা আছে। এটা git-এ push হলে exposed হবে। .gitignore চেক করুন, API key rotate করা বিবেচনা করুন।
